<p>這個月11/08 我又去考了一張AWS DVA了。DVA 的話會比較著墨在以開發者角度來看待 AWS 服務，比起 SAA 包山包海的內容來說，適合我們以 coding 出身的開發人員來考取。</p>



<!--more-->



<p>對於開發者來說不外乎 write code、testing、build、deploy 等軟體開發流程。故 DVA 會教導一系列為了協助開發者可以進行這個流程的 AWS 服務。</p>



<p>上課內容會重點在 AWS 的開發工具，比如像是 Serverless （Lambda, S3, API gateway, DynamoDB) 服務(大重點), Managed Service , IaC(Cloud formation, CDK), CI/CD Service , AWS SDK, CLI 之類的。</p>



<p>題目體感上算比 SAA 簡單，不外乎問最省錢、最省事、符合需求、最安全等要求。所以遇到題目類似備份，回答提到 AWS Backup, Manual Lambda + EventBridge, 如果看到很明顯用一堆服務又要手動的，可以先刪掉。</p>



<p>如果是資安掃描 S3 Bucket PII資訊等，答案有 Manual Lambda + s3 event triggered，基本上要優先選 Managed （Macie)。</p>



<p>我個人覺得是一張有認真上課 + 認證刷題後可以考取的一張證書。但它並不會測試你會不會寫程式、會不會資料結構、演算法等刷題能力，所以考過只能代表你對於開發者角度看待 AWS 是有一定的能力的。</p>
